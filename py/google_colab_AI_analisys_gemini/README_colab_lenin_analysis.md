# Анализ памятников Ленину с помощью Gemini (Google Colab)

## Назначение

Скрипт предназначен для автоматического анализа фотографий памятников Ленину с использованием мультимодальной модели Gemini. Для каждого изображения модель определяет набор заранее заданных признаков (тегов) и сохраняет результаты в CSV-файл.

Скрипт разработан для запуска в **Google Colab**.

---

## Предварительная подготовка

### 1. Клонирование репозитория

Перед запуском необходимо выполнить команду:

```python
!git clone https://github.com/spzw06/Lenin_in_moscow.git
```

После клонирования изображения должны находиться в каталоге:

```text
/content/Lenin_in_moscow/data/images
```

### 2. Настройка ключа Gemini API

Необходимо заранее создать и сохранить секрет:

```text
GEMINI_API_KEY
```

Скрипт пытается получить ключ через:

```python
userdata.get("GEMINI_API_KEY")
```

Если секрет не найден, выполнение будет остановлено.

## Основная логика работы

### 1. Поиск изображений

Скрипт сканирует каталог:

```text
/content/Lenin_in_moscow/data/images
```

и собирает все файлы форматов:

- `.png`
- `.jpg`
- `.jpeg`
- `.webp`

При необходимости количество обрабатываемых изображений можно ограничить параметром:

```python
MAX_IMAGES
```

---

### 2. Подготовка изображения

Для каждого файла:

1. Изображение открывается через Pillow.
2. Выполняется уменьшение размера до максимума 1500×1500 пикселей.
3. Изображение передаётся в Gemini вместе с текстовым промптом.

---

### 3. Анализ изображения моделью Gemini

Скрипт использует модель:

```python
gemini-3.1-flash-lite
```

(при необходимости можно заменить на другую совместимую модель).

Модель получает подробное описание ожидаемых признаков памятника и должна вернуть строго JSON-ответ.

Пример структуры ответа:

```json
{
  "стоит": true,
  "пальто": true,
  "в_кепке": false
}
```

---

### 4. Определяемые признаки

Скрипт анализирует следующие характеристики:

- тип памятника (бюст, поясной, в полный рост);
- поза (сидит, стоит);
- положение рук;
- наличие кепки;
- наличие пальто, костюма, галстука;
- наличие усов и бородки;
- наличие очков;
- книга или свиток в руках;
- скульптурная группа;
- отсутствие Ленина на фото;
- повреждения;
- следы вандализма;
- молодой или взрослый образ Ленина.

Всего используется фиксированный набор тегов из массива `EXPECTED_KEYS`.

---

### 5. Проверка и обработка ответа

После получения ответа:

1. Из текста извлекается JSON.
2. Выполняется очистка служебных символов.
3. JSON преобразуется в объект Python.
4. Значения тегов сопоставляются со списком ожидаемых полей.

Если модель вернула некорректный ответ, генерируется исключение.

---

### 6. Работа с лимитами Gemini

В скрипте реализована система повторных попыток.

При получении ошибки:

```text
429 RESOURCE_EXHAUSTED
```

выполняется:

- чтение рекомендуемого времени ожидания;
- автоматическая пауза;
- повторная отправка запроса.

Также отдельно обрабатывается ситуация исчерпания дневной квоты бесплатного тарифа.

---

### 7. Сохранение результатов

Результаты записываются в файл:

```text
results.csv
```

Для каждого изображения сохраняются:

- имя файла;
- значения всех тегов;
- текст ошибки (если анализ завершился неудачно).

Пример структуры CSV:

```text
filename,стоит,сидит,пальто,...,error
lenin_001.jpg,True,False,True,...,
```

Запись производится после обработки каждого изображения, что позволяет не потерять уже полученные данные при аварийном завершении работы.

---

## Защита от превышения лимитов

Скрипт учитывает ограничение модели по количеству запросов в минуту:

```python
MODEL_RPM = 15
```

На основании этого автоматически рассчитывается пауза между запросами:

```python
BASE_SLEEP_TIME = int(60 / MODEL_RPM)
```

Это снижает вероятность получения ошибок ограничения скорости.
